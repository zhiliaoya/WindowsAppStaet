# WindowsAppStaet

## 项目简介

**WindowsAppStaet** 是一款用于生成带优先级和延迟的 Windows 自动启动脚本的工具。它解决了 Windows 系统开机自启无序性的问题，特别适用于需要优先启动某一程序（如提供运行环境、代理服务等）作为其他程序运行必须依赖项的场景。

### 典型使用场景

1. **外贸工作场景**：优先启动沟通工具（如 WhatsApp、Skype、微信等），确保与客户沟通无障碍
2. **游戏场景**：在启动 Steam 之前优先启动游戏加速器，避免因加速器启动慢导致游戏连接失败
3. **开发环境**：优先启动数据库服务，再启动 IDE 或其他依赖数据库的应用程序
4. **企业办公**：优先启动 VPN 客户端，再启动企业办公软件

## 功能特性

- **拖拽添加**：支持将 `.lnk`（快捷方式）、`.url`（网址快捷方式）及其他文件直接拖拽到指定区域
- **单选勾选**：在列表中只能勾选一个项作为第一启动项（优先启动项）
- **可配置延迟**：设置第一启动项启动后等待的秒数（0-300秒），再启动其他程序
- **自动生成 BAT 脚本**：根据用户配置生成批处理启动脚本
- **自动添加到启动目录**：脚本生成后自动创建快捷方式到系统启动目录，实现开机自动运行
- **状态提示**：底部状态栏实时显示操作反馈

## 工作原理

1. 用户将需要自动启动的程序快捷方式拖拽到左侧区域
2. 在右侧列表中勾选需要优先启动的程序（只能勾选一个）
3. 设置优先启动后的等待时间
4. 点击「生成脚本」，程序将：
   - 生成一个 BAT 批处理脚本
   - 自动将该脚本的快捷方式添加到系统启动目录
5. 下次开机时，系统会按顺序自动启动所有程序

**生成的 BAT 脚本结构：**
```batch
@echo off
echo 正在启动第一程序：[优先启动的程序]
start "" "[程序路径]"
echo 第一程序已启动，等待X秒...
timeout /t X /nobreak >nul
echo 正在启动第2个程序：[程序B]
start "" "[程序B路径]"
...
```

## 安装与使用

### 方式一：直接运行（需要 Visual Studio）

1. 克隆本仓库或下载源代码
2. 使用 Visual Studio 2017 或更高版本打开 `WindowsAppStaet.sln`
3. 编译并运行项目

### 方式二：直接使用可执行文件

1. 从 [Releases](https://github.com/yourusername/WindowsAppStaet/releases) 页面下载最新版本的 `WindowsAppStaet.exe`
2. 双击运行即可（无需安装）

### 使用步骤

1. 将需要自动启动的程序快捷方式拖拽到左侧「拖拽快捷方式到这里」区域
2. 右侧列表会显示添加的程序名称，勾选需要优先启动的程序
3. 设置延迟时间（优先启动程序启动后等待多少秒再启动其他程序）
4. 点击「生成脚本」，确认保存位置
5. 程序会自动将生成的脚本快捷方式添加到系统启动目录

### 如何禁用自动启动？

- 按 `Win + R`，输入 `shell:startup`，打开启动目录
- 删除对应脚本的快捷方式即可

## 注意事项

- 请确保拖拽的是有效的 `.lnk`（快捷方式）文件，否则程序会直接使用原始路径
- 支持 `.url` 网址快捷方式，会解析其中的 URL 地址
- 生成的 BAT 脚本保存在您选择的位置，请勿随意移动或删除
- 如果无法自动创建启动快捷方式，可以手动将生成的 `.bat` 文件拖入启动目录（`shell:startup`）

### 为什么不用 Windows 计划任务？

计划任务的局限性（对普通用户不友好）

✅ 可视化顺序管理——哪个先启动、等多久，一目了然

✅ 一键生成 + 自启动——无需手动配置触发器

✅ 状态实时反馈——哪里出错了马上知道

✅ 适合非技术用户——外贸人员、游戏玩家、普通办公用户都能轻松上手

需要配置复杂的执行条件（如仅当网络可用时执行）

简单来说：计划任务是为管理员设计的，而 WindowsAppStaet 是为普通人设计的。
